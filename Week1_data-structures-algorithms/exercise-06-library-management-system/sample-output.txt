=== When to Use Each Algorithm ===
  Catalog size: 8 books
  Linear Search: O(n) — works on unsorted data, simple to implement.
  Binary Search: O(log n) — much faster, but requires sorted data.

  Use Linear Search when:
    - The catalog is small.
    - Books are added/removed frequently (sorting cost too high).
  Use Binary Search when:
    - The catalog is large and relatively stable.
    - Multiple searches are performed after each sort.
